# Generators

Os **Generators** recebem os dados estruturados produzidos pelos [Extractors](./extractors.md) e os transformam em arquivos Markdown de contexto escritos diretamente na instalação Moodle. São eles que tornam o conteúdo disponível para o servidor MCP servir via Resources e Tools.

---

## Visão geral

```
src/generators/
├── moodle.ts  ← 13 generators globais (raiz do Moodle)
└── plugin.ts  ← 11 generators por plugin (diretório do plugin)
```

Todos os generators seguem o mesmo contrato:
- Recebem um caminho (`moodlePath` ou `PluginInfo`) como entrada
- Chamam os extractors necessários
- Constroem o conteúdo Markdown em memória
- Consultam o [cache mtime](./cache-system.md) antes de escrever — se o arquivo `.md` já existe e é mais recente que os fontes PHP/XML, o arquivo é pulado (`skipped: true`)
- Escrevem o arquivo em disco via `writeFileSync`
- Retornam um `GeneratorResult` com `{ file, success, skipped?, error? }`
- Cada generator é envolvido em um **error boundary** via `safely()` — uma falha em um generator não interrompe os demais

## Generators globais (`moodle.ts`)

Geram arquivos na **raiz do Moodle**. Chamados por `init_moodle_context` e `update_indexes`.

A função de entrada é `generateAll(moodlePath, moodleVersion)` que executa todos os generators abaixo em paralelo.

| Função | Arquivo gerado | Extractors usados |

| `generateTasksIndex` | `MOODLE_TASKS_INDEX.md` | `tasks` |
| `generateServicesIndex` | `MOODLE_SERVICES_INDEX.md` | `services` |
| `generateDbTablesIndex` | `MOODLE_DB_TABLES_INDEX.md` | `schema` |
| `generateClassesIndex` | `MOODLE_CLASSES_INDEX.md` | `classes` |
| `generateCapabilitiesIndex` | `MOODLE_CAPABILITIES_INDEX.md` | `capabilities` |
| `generatePluginIndex` | `MOODLE_PLUGIN_INDEX.md` | `plugin` |
| `generateDevRules` | `MOODLE_DEV_RULES.md` | — (conteúdo estático) |
| `generatePluginGuide` | `MOODLE_PLUGIN_GUIDE.md` | — (conteúdo estático + versão) |

## Generators de plugin (`plugin.ts`)

Geram arquivos dentro do **diretório de cada plugin**. Chamados por `generate_plugin_context` e `plugin_batch`.

A função de entrada é `generateAllForPlugin(pluginPath, moodlePath, moodleVersion)` que executa todos os generators abaixo em sequência.

| Função | Arquivo gerado | Extractors usados |
|--------|---------------|-------------------|
| `generatePluginContext` | `PLUGIN_CONTEXT.md` | `plugin` |
| `generatePluginStructure` | `PLUGIN_STRUCTURE.md` | — (leitura de diretório) |
| `generatePluginDbTables` | `PLUGIN_DB_TABLES.md` | `schema` |
| `generatePluginEvents` | `PLUGIN_EVENTS.md` | `events` |
| `generatePluginDependencies` | `PLUGIN_DEPENDENCIES.md` | `tasks`, `services`, `capabilities`, `hooks`, `upgrade` |
| `generatePluginFunctionIndex` | `PLUGIN_FUNCTION_INDEX.md` | `api` (escopo do plugin) |
| `generatePluginCallbackIndex` | `PLUGIN_CALLBACK_INDEX.md` | `hooks` |
| `generatePluginEndpointIndex` | `PLUGIN_ENDPOINT_INDEX.md` | `services` |
| `generatePluginRuntimeFlow` | `PLUGIN_RUNTIME_FLOW.md` | `plugin`, `schema`, `events` |
| `generatePluginArchitecture` | `PLUGIN_ARCHITECTURE.md` | todos os anteriores |
| `generatePluginAiContext` | `PLUGIN_AI_CONTEXT.md` | lê os `.md` já gerados |

## Detalhes de design

### Error boundary — `safely()`

Todos os generators são envolvidos pela função `safely()`:

```typescript
async function safely(
  outputFile: string,
  fn: () => Promise<GeneratorResult>
): Promise<GeneratorResult> {
  try {
    return await fn();
  } catch (e) {
    return { file: outputFile, success: false, error: String(e) };
  }
}
```

Isso garante que se, por exemplo, `db/install.xml` estiver malformado, o `generatePluginDbTables` falha silenciosamente e os demais 10 generators do plugin continuam executando normalmente.

---

### Cabeçalho padrão — `header()`

Todos os arquivos `.md` gerados começam com um cabeçalho padronizado produzido pela função `header()`:

```
# Título do arquivo

> Descrição do conteúdo

_Generated by moodle-dev-mcp on 2024-04-22 10:30:00_

---
```

O timestamp permite detectar arquivos desatualizados sem precisar comparar conteúdo.

---

### Filtragem de arquivos gerados — `GENERATED_FILES`

O `generatePluginStructure` precisa listar o conteúdo do diretório do plugin sem incluir os próprios arquivos `.md` que o servidor gerou. Para isso, usa o conjunto `GENERATED_FILES`:

```typescript
const GENERATED_FILES = new Set([
  "PLUGIN_AI_CONTEXT.md",
  "PLUGIN_CONTEXT.md",
  "PLUGIN_STRUCTURE.md",
  // ... todos os 11 arquivos gerados
  ".moodle-mcp-dev",
]);
```

Arquivos presentes nesse conjunto são excluídos da árvore de diretórios mostrada no `PLUGIN_STRUCTURE.md`.

---

### `PLUGIN_AI_CONTEXT.md` — o consolidador

O `generatePluginAiContext` é sempre o **último** a ser executado na sequência de generators de plugin. Em vez de chamar extractors diretamente, ele **lê os arquivos `.md` já gerados pelos outros generators** e consolida as seções mais relevantes em um único arquivo otimizado para ser o ponto de entrada da IA.

Isso tem duas vantagens:
1. Não duplica lógica de extração — reutiliza o que já foi processado
2. O arquivo final reflete exatamente o estado atual dos demais arquivos

---

### Generators globais em paralelo vs. plugin em sequência

Os generators globais em `generateAll()` executam em **paralelo** — são independentes entre si e operam em arquivos diferentes, então não há risco de condição de corrida.

Os generators de plugin em `generateAllForPlugin()` executam em **sequência** — `generatePluginAiContext` precisa que todos os outros já tenham escrito seus arquivos antes de consolidá-los.

---

### `generateCtags` — geração opcional de ctags

O generator de ctags executa o comando `ctags` via `execSync` apenas se o binário `universal-ctags` estiver disponível no PATH. Se não estiver, retorna `skipped: true` sem erro. O arquivo `tags` gerado na raiz do Moodle permite navegação rápida de símbolos em editores compatíveis.

## Adicionando um novo generator

**1. Decida onde o arquivo gerado deve viver:**
- Raiz do Moodle → adicione em `moodle.ts` e registre em `generateAll()`
- Diretório do plugin → adicione em `plugin.ts` e registre em `generateAllForPlugin()`

**2. Implemente a função seguindo o contrato:**

```typescript
export async function generateMeuArquivo(
  moodlePath: string
): Promise<GeneratorResult> {
  return safely(join(moodlePath, 'MEU_ARQUIVO.md'), async () => {
    // 1. Verificar cache
    const sourceFiles = [join(moodlePath, 'lib', 'meuarquivo.php')];
    if (globalCache.isUpToDate(join(moodlePath, 'MEU_ARQUIVO.md'), sourceFiles)) {
      return { file: join(moodlePath, 'MEU_ARQUIVO.md'), success: true, skipped: true };
    }

    // 2. Chamar extractor(es)
    const dados = extractMeuDado(join(moodlePath, 'lib', 'meuarquivo.php'));

    // 3. Construir conteúdo Markdown
    const content = [
      header('Meu Arquivo', 'Descrição do conteúdo'),
      dados.map(d => `- ${d.campo}`).join('\n'),
    ].join('\n');

    // 4. Escrever em disco
    return write(join(moodlePath, 'MEU_ARQUIVO.md'), content);
  });
}
```

**3. Registre em `generateAll()` ou `generateAllForPlugin()`:**

```typescript
// Em generateAll():
results.push(await safely(..., () => generateMeuArquivo(moodlePath)));
```

**4. Adicione a entrada ao `GENERATED_FILES` se for arquivo de plugin.**

**5. Documente o novo arquivo em [Arquivos Gerados](../reference/generated-files.md).**
