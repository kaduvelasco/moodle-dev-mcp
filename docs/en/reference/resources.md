# Resources Reference (MCP)

**Resources** function as a reading library for your AI assistant. They provide structured data about the Moodle installation and its plugins through custom URIs тАФ without requiring an explicit call from the user.

## How the AI Uses Resources

The AI automatically decides when to read a resource based on your question. But you can be explicit to guide the query:

- _"Take a look at the resource `moodle://api-index` and see if there is a function to delete users in bulk."_
- _"Analyze the schema in `moodle://plugin/local_myplugin/database` and suggest missing indexes."_
- _"Before suggesting fixes, check the standards in `moodle://dev-rules`."_

Resources are generated by the server Generators and remain available as long as the `.md` context files exist in the installation. Use `init_moodle_context` or `generate_plugin_context` to generate or update the content.

## ЁЯМН Global Resources

These resources provide a panoramic view of the entire Moodle installation.

| URI                             | Description                                                               |
| :------------------------------ | :------------------------------------------------------------------------ |
| `moodle://context`              | Installation overview, detected version, and coding guidelines            |
| `moodle://index`                | Master index of all generated context files                               |
| `moodle://workspace`            | Guide for the AI on how to navigate and work within the current workspace |
| `moodle://api-index`            | Catalog of all public and deprecated functions from the core `lib/`       |
| `moodle://classes-index`        | All PHP classes, interfaces, and traits in the installation               |
| `moodle://db-tables`            | Complete structure of all database tables                                 |
| `moodle://plugin-index`         | Map of all installed plugins with versions and paths                      |
| `moodle://capabilities-index`   | All capabilities (permissions) available in the system                    |
| `moodle://events-index`         | Event observers registered across all plugins                             |
| `moodle://tasks-index`          | All scheduled tasks in the installation                                   |
| `moodle://services-index`       | All registered web service functions                                      |
| `moodle://dev-rules`            | Moodle coding standards (Moodle Coding Style)                             |
| `moodle://plugin-guide`         | Quick reference of structure by plugin type                               |
| `moodle://plugins/with-context` | List of plugins that already have AI context generated                    |

## ЁЯзй Plugin Resources

Resources specific to each plugin. Replace `{component}` with the plugin frankenstyle тАФ for example: `local_myplugin`, `mod_assign`, `block_html`.

### Entry point

| URI                                   | Description                                                            |
| :------------------------------------ | :--------------------------------------------------------------------- |
| `moodle://plugin/{component}`         | Complete plugin context тАФ recommended entry point to start any session |
| `moodle://plugin/{component}/context` | Plugin metadata: component, type, version, and path                    |

### Structure and code

| URI                                        | Description                                      |
| :----------------------------------------- | :----------------------------------------------- |
| `moodle://plugin/{component}/structure`    | Plugin directory tree (generated files excluded) |
| `moodle://plugin/{component}/architecture` | Architecture overview and design notes           |
| `moodle://plugin/{component}/functions`    | All PHP functions with file and line references  |

### Database

| URI                                    | Description                                                              |
| :------------------------------------- | :----------------------------------------------------------------------- |
| `moodle://plugin/{component}/database` | Complete schema of tables (`db/install.xml`): fields, types, and indexes |

### Integrations and extensibility

| URI                                        | Description                                                       |
| :----------------------------------------- | :---------------------------------------------------------------- |
| `moodle://plugin/{component}/events`       | Event observers registered in `db/events.php`                     |
| `moodle://plugin/{component}/callbacks`    | Legacy callbacks from `lib.php` and Hook API registrations (4.3+) |
| `moodle://plugin/{component}/endpoints`    | Web services and AJAX endpoints defined by the plugin             |
| `moodle://plugin/{component}/dependencies` | Tasks, services, capabilities, and upgrade history                |

### Execution flow

| URI                                | Description                            |
| :--------------------------------- | :------------------------------------- |
| `moodle://plugin/{component}/flow` | Entry points and plugin execution flow |
