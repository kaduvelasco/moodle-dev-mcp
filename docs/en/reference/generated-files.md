# Generated Context Files

`moodle-dev-mcp` generates structured Markdown files directly in your Moodle installation. These files serve as a "knowledge bridge" — allowing the AI assistant to understand your code architecture without needing to read thousands of lines of raw PHP.

The files are generated by the server tools and kept up to date by the mtime cache system: a file is only rewritten if the corresponding PHP or XML source code has been modified since the last run.

---

## 📂 Global Files (Moodle root)

Generated by the `init_moodle_context` and `update_indexes` tools. They map the entire installation.

| File                           | Description                                                                          |
| :----------------------------- | :----------------------------------------------------------------------------------- |
| `AI_CONTEXT.md`                | Installation summary: version, paths, and development guidelines                     |
| `MOODLE_AI_INDEX.md`           | Master index of all generated context files                                          |
| `MOODLE_AI_WORKSPACE.md`       | Guide for the AI on how to navigate and work within the workspace                    |
| `MOODLE_API_INDEX.md`          | Catalog of core functions with signatures, visibility, and `@since`                  |
| `MOODLE_CLASSES_INDEX.md`      | Index of PHP classes, interfaces, and traits                                         |
| `MOODLE_CAPABILITIES_INDEX.md` | All capabilities (permissions) available in the installation                         |
| `MOODLE_DB_TABLES_INDEX.md`    | Complete structure of all `mdl_` tables                                              |
| `MOODLE_EVENTS_INDEX.md`       | All event observers registered in the installation                                   |
| `MOODLE_PLUGIN_INDEX.md`       | Map of all installed plugins with versions and paths                                 |
| `MOODLE_SERVICES_INDEX.md`     | All registered web service functions                                                 |
| `MOODLE_TASKS_INDEX.md`        | All scheduled tasks and ad-hoc tasks                                                 |
| `MOODLE_DEV_RULES.md`          | Moodle coding standards (Moodle Coding Style) for AI reference                       |
| `MOODLE_PLUGIN_GUIDE.md`       | Quick reference of structure by plugin type                                          |
| `tags`                         | ctags file for quick symbol navigation (generated if `universal-ctags` is installed) |

## 🧩 Files per Plugin

Generated by the `generate_plugin_context` tool (or `plugin_batch`). Created inside each plugin directory — for example: `local/myplugin/`.

### Main context

| File                   | Description                                                                                               |
| :--------------------- | :-------------------------------------------------------------------------------------------------------- |
| `PLUGIN_AI_CONTEXT.md` | **Main file.** Consolidates the most relevant plugin context — load this first in any development session |
| `PLUGIN_CONTEXT.md`    | Plugin metadata: component, type, version, path, and declared dependencies                                |

### Structure and architecture

| File                     | Description                                            |
| :----------------------- | :----------------------------------------------------- |
| `PLUGIN_STRUCTURE.md`    | Plugin directory tree (`.md` generated files excluded) |
| `PLUGIN_ARCHITECTURE.md` | Architecture overview and design notes                 |
| `PLUGIN_RUNTIME_FLOW.md` | Entry points and plugin execution flow                 |

### Database

| File                  | Description                                                                         |
| :-------------------- | :---------------------------------------------------------------------------------- |
| `PLUGIN_DB_TABLES.md` | Complete schema extracted from `db/install.xml`: tables, fields, types, and indexes |

### Code and functions

| File                       | Description                                                              |
| :------------------------- | :----------------------------------------------------------------------- |
| `PLUGIN_FUNCTION_INDEX.md` | All plugin PHP functions with file, line, and signature                  |
| `PLUGIN_CALLBACK_INDEX.md` | Legacy callbacks from `lib.php` and Hook API registrations (Moodle 4.3+) |

### Integrations

| File                       | Description                                                      |
| :------------------------- | :--------------------------------------------------------------- |
| `PLUGIN_EVENTS.md`         | Event observers registered in `db/events.php`                    |
| `PLUGIN_ENDPOINT_INDEX.md` | Web services and AJAX endpoints defined by the plugin            |
| `PLUGIN_DEPENDENCIES.md`   | Scheduled tasks, capabilities, web services, and upgrade history |

## ❓ Frequently Asked Questions

### Should I version these files in Git?

**Not recommended.** These files are generated automatically and change whenever the source code is updated. Add them to the project `.gitignore`:

```gitignore
# moodle-dev-mcp context files
AI_CONTEXT.md
MOODLE_AI_*.md
MOODLE_*_INDEX.md
MOODLE_DEV_RULES.md
MOODLE_PLUGIN_GUIDE.md
PLUGIN_AI_*.md
PLUGIN_*_INDEX.md
PLUGIN_STRUCTURE.md
PLUGIN_DB_TABLES.md
PLUGIN_EVENTS.md
PLUGIN_ARCHITECTURE.md
PLUGIN_RUNTIME_FLOW.md
.moodle-mcp
```

### Do these files add weight to the server?

No. They are simple Markdown text files, usually only a few KB each. The mtime cache system ensures that only files whose PHP or XML sources were modified are rewritten — subsequent runs are fast.

### Can I edit these files manually?

Not recommended. Any manual edit will be overwritten the next time the server regenerates the context. To add persistent notes for the AI about a project or plugin, use `CLAUDE.md` (Claude Code) or `GEMINI.md` (Gemini Code Assist) in the installation root. These files are automatically read by the AI client in every session and are never touched by the server.

### Why wasn’t a file updated after I changed the code?

The cache system compares the `mtime` (modification date) of the PHP file with the corresponding `.md` file. If the `.md` file is newer, it is skipped. To force full regeneration, ask the assistant: _"Regenerate the context of plugin local_myplugin ignoring the cache."_ The AI will call `generate_plugin_context` with `force=true`.
